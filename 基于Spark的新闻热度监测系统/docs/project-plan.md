# 项目总规划

## 1. 项目定位

本项目不追求任务书中“全媒体实时舆情平台”的产品级规模，而是围绕毕业设计的实际周期，收敛为一个可运行、可展示、可答辩的新闻热度监测系统 MVP。系统重点在于证明以下技术链路能够闭环：

1. 能稳定采集真实新闻数据
2. 能完成清洗、去重和结构化入库
3. 能用 Spark 完成批处理分析
4. 能输出热点、趋势、情感、事件等可解释结果
5. 能通过前端大屏和查询导出把分析结果展示出来

## 2. 总体架构

系统按六层设计：

1. 数据源接入层
   - 公开数据集导入
   - RSS/API 采集
   - 网页正文解析
2. 原始数据层
   - 原始新闻 JSON/CSV 存档
   - 采集日志和来源记录
3. 清洗去重层
   - HTML 清洗
   - 文本标准化
   - 时间格式归一化
   - URL/标题哈希强去重
   - SimHash + 标题相似度弱去重
4. Spark 批处理分析层
   - 热度指数计算
   - 关键词趋势统计
   - 基础情感分析
   - 基础事件聚类
   - 规则预警
5. 后端服务层
   - REST API
   - 数据查询筛选
   - 数据导出
6. 前端展示层
   - 可视化大屏
   - 新闻查询
   - 事件分析

## 3. MVP 核心功能

### 必做功能

- 新闻采集
- 数据清洗与去重
- 本地 Spark 批处理
- 热度指数计算
- 关键词趋势统计
- 基础情感分析
- 基础事件聚类
- 可视化大屏
- 查询、筛选、导出

### 可作为后续扩展的功能

- 更高质量的中文情感模型微调
- 更精细的事件聚类与命名
- 多角色服务界面
- 复杂传播评估
- 流式处理与实时预警

## 4. 技术路线

### 后端与分析

- 语言：Python 3.11
- Web：Flask
- 数据分析：PySpark + Pandas
- 机器学习：scikit-learn
- 文本处理：jieba、BeautifulSoup、feedparser
- 数据库存储：SQLite 默认，兼容 MySQL
- 中间结果：Parquet / JSON

### 前端

- Vue 3
- Vite
- Axios
- ECharts

## 5. 数据方案

采用混合数据策略：

1. 公开新闻数据集作为基础语料，保证样本规模
2. RSS/API 采集作为真实动态数据来源，保证真实性和时效性
3. 网页解析作为补充采集方式，覆盖无标准接口的网站

目标是在交付前形成不少于 10000 条结构化新闻数据。

## 6. 实施顺序

### 阶段一：数据底座

- 修正本地 Spark 环境
- 建立新闻标准结构
- 打通采集、清洗、去重链路
- 建立原始数据与清洗数据存储

### 阶段二：分析链路

- 基于清洗数据完成热点与趋势统计
- 落地基础情感分析
- 落地基础事件聚类
- 输出预警规则结果

### 阶段三：系统展示

- 提供后端 API
- 搭建大屏、查询页、事件页
- 完成筛选和导出

### 阶段四：优化与论文支撑

- 补充真实采集样本
- 调整热度权重与聚类参数
- 输出测试结果、图表和答辩材料

## 7. 项目风险与应对

### 风险一：采集源不稳定

- 采用多来源组合，避免单一网站失效
- 保留本地样例数据，确保演示和联调不中断

### 风险二：本地 Spark 环境不完整

- 将 PySpark 作为首选
- 在环境异常时提供 Pandas 退化方案，确保核心功能可演示

### 风险三：模型效果不足

- 情感分析先采用词典/公开模型基线
- 事件聚类先做基础版本，后续通过小规模补标优化

## 8. 答辩表达主线

答辩时重点强调：

1. 这个项目最难的不是图表，而是高质量新闻数据采集与清洗
2. 先把数据底座做好，后续热度、情感、聚类才有可信度
3. 采用 Spark 的原因是面向批量新闻处理和后续扩展，而不是为了堆技术名词
4. 第一版先解决“能用、可解释、可演示”，复杂功能作为后续研究方向
