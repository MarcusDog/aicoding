# 系统优化清单

本文档基于当前 MVP 代码整理，目的是明确哪些部分已经跑通，哪些部分仍然只是基础版，以及下一步该怎样增强。你后面写论文“系统优化方向”或答辩“后续工作”时，可以直接引用这份清单。

## 数据采集层

当前状态：
已支持样例数据、批量导入、RSS 采集和基础网页抓取；真实新闻源配置已外置到 `backend/config/source_catalog.json`；可批量导入 `csv/json/jsonl`。

需要优化：
- 增加更多稳定中文 RSS 源，并维护来源分类、地区、语言和启用状态
- 增加采集失败日志、状态码记录、重试策略和采集耗时统计
- 为采集结果记录批次号、任务号和导入文件名，便于追踪
- 网页采集只做了非常基础的正文提取，需要增强主内容识别、发布时间提取和编码处理
- 增加增量采集策略，避免重复抓取
- 增加来源健康检查和“最近成功采集时间”统计

优先级：
- 高：稳定 RSS 源、采集日志、失败重试
- 中：网页正文提取增强、增量采集

## 数据清洗与去重层

当前状态：
已支持 HTML 清洗、时间归一化、关键词抽取，以及 URL/标题哈希强去重和 SimHash/标题相似度弱去重。

需要优化：
- 当前去重阈值是经验值，缺少人工标注样本验证误杀率和漏判率
- 内容摘要生成较简单，长文本截断会损失上下文
- 缺少字段完整率、乱码率、正文提取成功率等质量指标
- 缺少黑名单词、广告模板、导航噪声的更细粒度过滤
- 需要补统一停用词表、媒体别名映射表和栏目标准化规则

优先级：
- 高：质量统计、去重评估样本、来源标准化
- 中：摘要生成优化、广告噪声规则库

## 热度计算层

当前状态：
已实现基础可解释热度模型，综合考虑来源数量、来源权重、情感强度和时效衰减。

需要优化：
- 当前热度权重是手工设定，缺少实验对比和参数依据
- 媒体权威度映射很少，覆盖不足
- 没有引入互动指标、转载层级、评论量等更强信号
- 缺少“事件级热度”和“新闻级热度”的对比
- 缺少不同场景下可切换的权重方案

优先级：
- 高：扩展来源权威度映射、记录热度公式版本
- 中：事件级热度、场景化权重

## 关键词趋势层

当前状态：
已支持基础关键词抽取与按小时聚合，并能输出趋势数据到大屏。

需要优化：
- 当前主要是词频驱动，缺少更稳健的 TF-IDF / TextRank / 新词发现
- 缺少按日、按来源、按类别、按地区的多维趋势查询
- 缺少关键词合并规则，例如“人工智能”和“AI”
- 缺少停用词、行业词典和热词白名单维护机制

优先级：
- 高：词典维护和同义词合并
- 中：多维趋势统计

## 情感分析层

当前状态：
当前仍是关键词词典规则法，只能作为基线，已能输出正面、中性、负面三分类。

需要优化：
- 规则法精度较低，尤其对新闻报道类文本容易误判
- 缺少公开中文情感数据集训练与评估流程
- 缺少人工补标新闻样本和验证集
- 缺少按来源、时间、事件簇聚合的情感变化曲线
- 缺少模型版本管理和基线对比

优先级：
- 高：接入公开中文情感模型，建立验证集
- 中：补充新闻领域小规模标注和微调

## 事件聚类层

当前状态：
当前采用 TF-IDF + 层次聚类，已能输出基础事件簇、代表标题和关键词标签。

需要优化：
- 当前聚类主要依赖标题和摘要，语义能力有限
- 聚类簇数策略较简单，对不同规模数据不稳定
- 缺少聚类质量评估指标和人工抽样验证
- 缺少事件命名策略优化，标签可读性还不够强
- 缺少事件演化时间线和同事件跨来源传播展示

优先级：
- 高：中文句向量模型替换、聚类质量评估
- 中：事件命名优化、时间线展示

## 预警层

当前状态：
已实现简单规则预警，目前仅支持新闻量突增和负面占比偏高。

需要优化：
- 规则较少，误报与漏报不可控
- 缺少关键词异常增长、来源覆盖突增、区域扩散等预警维度
- 缺少预警历史、确认状态和人工复核机制
- 缺少不同级别预警阈值配置

优先级：
- 高：阈值配置、预警历史
- 中：更多预警规则

## 后端 API 层

当前状态：
已具备大屏、查询、事件、导出和流水线触发接口。

需要优化：
- 查询筛选条件还不够细，缺少时间范围、类别、地区、热度区间过滤
- 导出接口当前是全量导出，应支持按当前筛选条件导出
- 缺少统一错误码、参数校验和异常响应格式
- 缺少接口文档和示例请求
- 缺少采集任务、来源管理、统计报表相关接口

优先级：
- 高：筛选导出联动、参数校验
- 中：接口文档、任务接口

## 前端展示层

当前状态：
已有大屏、新闻查询页、事件分析页，能展示热点、趋势、情感和事件结果。

需要优化：
- 当前页面仍偏 MVP，缺少采集任务管理、数据质量看板、来源配置页
- 缺少加载态、错误态和空数据态处理
- 查询页筛选项不够丰富
- 大屏图表联动较少，无法从热点直接钻取新闻详情
- 当前页面缺少系统截图友好布局，不利于论文插图

优先级：
- 高：筛选增强、图表联动、页面状态完善
- 中：采集管理页、截图友好布局

## 存储与任务运行层

当前状态：
默认 SQLite，可兼容 MySQL；分析结果已保存到 Parquet / JSON / 数据库。

需要优化：
- 缺少采集批次表、任务运行日志表和来源状态表
- 缺少任务历史记录和失败原因留档
- 缺少定时运行机制
- 缺少冷热数据分层和历史归档策略

优先级：
- 高：任务日志表、批次表
- 中：定时任务和历史归档

## 测试与质量保障

当前状态：
已有基础测试，覆盖去重、分析输出和批量导入。

需要优化：
- 测试覆盖面还不够，尤其是 API、批量导入、导出和前端页面
- 缺少针对真实新闻样本的回归测试
- 缺少性能测试和数据量扩大后的验证
- 缺少人工抽样评估表，无法为论文测试章节提供充分证据

优先级：
- 高：API 测试、导入导出测试、样本回归测试
- 中：性能测试和人工评估表

## 当前建议的优化顺序

1. 修复所有中文乱码和界面文案问题
2. 增强真实采集源、采集日志和来源健康检查
3. 完善查询筛选、导出联动和采集统计接口
4. 把情感分析从规则法升级到公开中文模型
5. 把事件聚类从 TF-IDF 升级到中文句向量
6. 增加采集管理页、数据质量看板和任务历史
7. 补齐论文所需测试数据、截图和对比表
