# 中期检查报告格式规范

## 1. 依据文件

- 模板主文件：`thesis/docs/软件学院毕业设计说明书格式规范+表格模板/2025届软件学院毕业设计说明书格式规范/毕业设计中期检查表2023.doc`
- 案例参考：`thesis/docs/中期检查报告案例文档.docx`
- 补充要求：用户明确要求“二级标题与正文宋体小四，标题顶格，正文行间距 1.5 倍且段落首行缩进；所有图片居中且配有图题，图题宋体小四居中”

## 2. 版式结论

- 页面边距：上约 25.4 mm，下约 25.4 mm，左约 31.8 mm，右约 31.8 mm。
- 文档标题：沿用模板现有标题“中北大学软件学院毕业设计中期报告”，居中排版，不改标题层级结构。
- 主体结构：标题下方为 8 行表格，内容主要写入第 4、5、6 行的大单元格。
- 教师评语：第 7 行属于指导教师填写区域，学生版文档不编造评价内容。

## 3. 表格槽位

### 3.1 基本信息区

- 第 1 行：专业、方向、班级
- 第 2 行：姓名、学号、指导教师
- 第 3 行：设计题目

### 3.2 学生填写正文区

- 第 4 行：`本人在该设计中具体应完成的工作`
- 第 5 行：`简述毕业设计开始以来所做的具体工作和取得的进展（要详细内容）`
- 第 6 行：`目前存在问题，下一步的主要设计任务，具体设想与安排（要详细内容）`

## 4. 字体与段落执行规则

- 表格正文统一使用宋体、小四（12 pt）。
- 二级标题与正文同为宋体、小四。
- 标题顶格，不做首行缩进。
- 正文段落首行缩进 2 字符。
- 正文行距统一为 1.5 倍。
- 表格内正文默认左对齐。
- 表格标签类单元格保持模板原有居中或对齐方式，不主动重排。

## 5. 图片规则

- 中期报告如插入图片，图片必须居中。
- 图片下方必须有图题。
- 图题格式：宋体、小四、居中。
- 图片与图题不得拆散；若空间不足，应整体下移到下一页或下一段落区域。

## 6. 内容边界

- 只写“中期已完成情况”和“下一步安排”，不扩写成完整毕业论文。
- 内容应基于项目当前真实完成情况，包括已实现模块、已形成的数据结果、已完成测试和当前遗留问题。
- 不夸大为“全部完成”或“正式交付”，要体现中期检查语境。

## 7. 本次执行口径

- 项目题目采用任务书中的正式题目：`基于Spark的新闻热度监测系统的设计与实现`。
- 自动可确认的信息直接填入模板；无法从仓库可靠获取的个人信息字段保留空位，避免虚构。
- 教师评语栏保留模板区域，供导师后续填写。
