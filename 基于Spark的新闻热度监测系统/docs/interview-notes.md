# 复试/答辩准备

## 1. 用一句话介绍项目

这是一个基于 Python、Spark、Flask 和 Vue 构建的新闻热度监测系统，重点解决新闻数据采集、清洗去重、热点分析、情感分析、事件聚类和可视化展示问题。

## 2. 你做这个项目的核心思路

我的思路不是先堆算法，而是先保证新闻数据质量。因为新闻热度监测的本质是“数据驱动判断”，如果原始数据质量差，后面的热度指数、情感分析和事件聚类都会受到影响。所以我先做采集、清洗和去重，再做分析，最后做可视化和导出。

## 3. 为什么这样规划

### 原因一：任务书范围很大

任务书包含了实时采集、热度分析、聚类、情感、传播评估、预警、多角色服务等大量内容。如果不做范围收敛，项目容易停留在概念层面，无法落地。

### 原因二：毕业设计需要可交付

毕业设计最重要的是在规定周期内做出完整闭环，包括系统功能、测试结果、论文章节和答辩表达。因此我把范围缩小到 MVP，把最关键、最能体现技术含量的部分先做出来。

### 原因三：采集决定系统上限

新闻监测系统的结果可信度取决于采集数据的真实性、完整性和去重质量，这部分决定系统上限。

## 4. 如果老师问你有哪些难点

可以回答以下几类难点：

1. 多来源新闻结构不统一，字段清洗成本高
2. 新闻转载严重，去重策略必须兼顾准确率和效率
3. 本地 Spark 环境要兼顾可跑通和后续扩展
4. 情感分析和事件聚类要在有限时间内做到“可用且可解释”
5. 前端不仅要展示图表，还要支持查询、筛选和导出

## 5. 如果老师问你后续怎么优化

可以回答：

1. 扩展更多真实新闻源和更稳定的增量采集策略
2. 引入中文预训练模型提升情感分析精度
3. 用句向量和更细粒度特征增强事件聚类效果
4. 将本地 Spark 批处理扩展为流批一体的准实时处理
5. 增强预警规则和多角色服务能力

## 6. 现阶段你最应该准备的内容

1. 讲清楚为什么先做数据采集与清洗
2. 讲清楚 Spark 在这里的作用
3. 讲清楚热度指数的设计逻辑
4. 讲清楚情感分析和聚类为什么先做基础版
5. 讲清楚整个系统的数据流和模块关系

## 7. 简短回答模板

### 题目背景

“新闻数据量大、传播快、转载多，人工监测效率低，所以需要构建一个能够自动采集、清洗、分析和展示新闻热点的系统。”

### 技术路线

“我采用 Python 负责采集和后端服务，使用 Spark 处理批量新闻数据，通过 Flask 提供接口，再用 Vue 和 ECharts 做可视化展示。”

### 核心难点

“最大的难点其实是数据底座建设，尤其是多来源新闻清洗和去重，因为它直接决定后续分析的质量。”

### 后续工作

“当前先把可运行的 MVP 做出来，后续会继续补充数据规模、优化情感模型和聚类策略，并增强预警与传播分析能力。”
