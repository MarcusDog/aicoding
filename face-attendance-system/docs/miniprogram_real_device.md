# 微信小程序真机测试（iOS + Android）

## 1. 前置准备

- 手机与后端机器在同一局域网
- 后端服务已启动并能被手机访问
- `miniprogram/config/index.js` 中 `dev.baseURL` 改为电脑局域网 IP
  - 示例：`http://192.168.1.20:5000/api`

## 2. 开发者工具设置

- 上传代码前先在开发者工具中勾选：
  - 不校验合法域名
  - 不校验 HTTPS 证书（仅开发环境）
- 真机调试时打开“调试基础库”

## 3. iOS 实测清单

- 首次进入签到页是否能弹出定位授权
- 摄像头是否能正常预览与拍照
- 拍照后是否能压缩上传并返回签到结果
- 弱网下是否能自动重试请求（request retry）

## 4. Android 实测清单

- 定位授权后是否能稳定获取经纬度
- 不同机型相机方向/分辨率下上传是否成功
- 系统返回的机型信息是否写入 `device_info`

## 5. 线上配置注意

- 生产环境必须配置合法 `request` 域名
- 建议使用 HTTPS 域名（避免网络层拦截）
- `WECHAT_LOGIN_MODE` 改为 `real`，并配置 `APPID/SECRET`

## 6. 排障建议

- 报错“定位失败”：检查系统定位权限和小程序授权状态
- 报错“网络请求失败”：先在手机浏览器访问后端健康检查地址
- 报错“人脸质量低”：提高环境光，避免逆光与模糊
