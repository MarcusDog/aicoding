# 测试报告

## 1. 测试目标

验证基于 WEB 的家教信息发布和求职平台是否满足任务书要求，重点覆盖家长发布需求、管理员审核、教员申请、订单生成和统计分析等核心流程。

## 2. 测试范围

- 后端服务层业务测试：验证状态流转和异常约束。
- Vue 前端静态资源测试：验证 Spring Boot 可以托管 Vue 运行时、Vue Router 和前端入口页，并能将前端路径转发到入口页。
- Python 分析模块测试：验证统计指标和运营建议。
- 兼容性测试：建议使用 Chrome、Edge、Safari 等主流浏览器进行手工验证。

## 3. 主要用例

| 编号 | 用例 | 预期结果 |
| --- | --- | --- |
| TC-01 | 家长发布需求 | 需求状态为待审核 |
| TC-02 | 教员申请未审核需求 | 系统拒绝申请 |
| TC-03 | 管理员审核需求 | 需求状态变为开放中 |
| TC-04 | 未认证教员申请需求 | 系统提示需先认证 |
| TC-05 | 管理员通过申请 | 系统生成服务订单 |
| TC-06 | 前端切换三类角色 | 页面展示对应工作区 |
| TC-07 | Python 统计分析 | 输出角色、需求、申请和订单统计 |
| TC-08 | 教员资料被驳回后修改并重新提交 | 资料状态恢复为待审核，原驳回理由清空 |
| TC-09 | 家长需求被驳回后修改并重新提交 | 需求状态恢复为待审核，原驳回理由清空 |
| TC-10 | 前台详情页查询 | 可查看需求详情和教员资料详情 |

## 4. 测试命令

```bash
cd backend
mvn test
```

```bash
python3 -m unittest discover -s tests
```

## 5. 环境说明

后端测试和运行需要安装 JDK 17+ 与 Maven 3.9+，前端资源同步需要 Node.js 20+。当前工程默认使用 H2 内存数据库便于开发测试，使用 `mysql` profile 可连接 MySQL。

## 6. 当前验证结果

- 后端 `mvn test` 已通过，服务层共 6 个测试用例，覆盖审核、接单、详情查询和驳回后重新提交。
- 前端静态资源测试已通过，`/index.html` 可加载 Vue 运行时、Vue Router 与前端入口，`/`、`/admin` 等前端路径可转发到入口页。
- 已通过 HTTP 请求手工验证 `PUT /api/tutors/{tutorId}/profile` 和 `PUT /api/parents/{parentId}/demands/{demandId}` 两个重新提交接口。
