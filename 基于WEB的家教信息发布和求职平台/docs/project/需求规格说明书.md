# 基于 WEB 的家教信息发布和求职平台需求规格说明书

## 1. 项目背景

传统家教市场存在信息分散、身份真实性难核验、需求响应慢、订单管理不规范等问题。平台以家长、教员、管理员三类角色为核心，通过线上发布、审核、申请、匹配和统计功能，提高家教供需匹配效率。

## 2. 用户角色

- 家长：注册登录后发布家教需求，查看需求审核状态和订单状态。
- 教员：提交个人资质信息，认证通过后浏览开放需求并提交接单申请。
- 管理员：审核教员认证、审核家教需求、处理接单申请、查看系统统计和审核记录。

## 3. 功能需求

- 用户管理：支持演示登录、角色识别、账号状态维护。
- 教员认证：教员提交学校、专业、擅长科目和简介，管理员审核后才能接单。
- 需求发布：家长填写标题、科目、年级、地点、预算、时间和描述，提交后进入待审核状态。
- 需求浏览：审核通过的需求向教员开放。
- 接单申请：认证教员可对开放需求提交申请，系统防止重复申请。
- 申请审核：管理员通过申请后，系统自动生成订单并将需求标记为已匹配。
- 统计分析：统计角色数量、开放需求、待审核资料、待处理申请和服务订单数量。

## 4. 非功能需求

- 易用性：Vue 前端按角色组织页面与路由，降低操作复杂度。
- 可维护性：后端采用实体、仓储、服务、控制器分层。
- 数据一致性：核心业务状态变更集中在服务层事务中完成。
- 可测试性：后端服务、前端组件和 Python 统计模块均提供测试入口。

## 5. 业务流程

1. 家长发布需求，需求状态为 `PENDING_REVIEW`。
2. 管理员审核需求，通过后状态变为 `OPEN`。
3. 教员提交认证资料，通过后状态变为 `APPROVED`。
4. 认证教员申请开放需求，申请状态为 `SUBMITTED`。
5. 管理员通过申请，申请状态变为 `ACCEPTED`，需求状态变为 `MATCHED`，订单状态为 `ACTIVE`。
