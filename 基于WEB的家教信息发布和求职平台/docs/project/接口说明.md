# 接口说明

## 1. 认证接口

### `POST /api/auth/login`

请求：

```json
{
  "username": "parent",
  "password": "123456"
}
```

返回用户信息，包含 `id`、`username`、`displayName`、`role` 等字段。

### `POST /api/auth/register`

注册家长或教员账号。管理员账号不开放前台注册。

请求：

```json
{
  "username": "new_parent",
  "password": "123456",
  "displayName": "王家长",
  "phone": "18800000000",
  "role": "PARENT"
}
```

## 2. 家长端接口

### `POST /api/parents/{parentId}/demands`

发布家教需求。需求提交后默认进入 `PENDING_REVIEW`。

### `PUT /api/parents/{parentId}/demands/{demandId}`

修改被管理员驳回的家教需求并重新提交。重新提交后需求状态恢复为 `PENDING_REVIEW`，原审核备注清空，等待管理员复审。

### `GET /api/parents/{parentId}/demands`

查询家长发布的需求列表。

### `GET /api/parents/{parentId}/orders`

查询家长侧服务订单列表。

## 3. 教员端接口

### `POST /api/tutors/{tutorId}/profile`

提交教员认证资料，包含学校、专业、可授科目、个人简介、教学经验、服务方式和文字简历。

### `PUT /api/tutors/{tutorId}/profile`

修改被管理员驳回的教员认证资料并重新提交。重新提交后资料状态恢复为 `PENDING`，原审核备注清空。

### `POST /api/tutors/{tutorId}/profile/resume-file`

上传教员简历附件，支持 `pdf`、`doc`、`docx`、`txt` 格式。

### `GET /api/tutors/{tutorId}/profile`

查询教员自己的认证资料。

### `GET /api/tutors/demands/open`

查询当前开放可申请的需求。

### `POST /api/tutors/{tutorId}/applications`

教员对指定需求提交接单申请。

### `GET /api/tutors/{tutorId}/applications`

查询教员自己的申请记录。

### `GET /api/tutors/{tutorId}/orders`

查询教员侧服务订单列表。

## 4. 公共展示接口

### `GET /api/catalog/demands/open`

查询前台展示的开放家教需求列表。

### `GET /api/catalog/demands/{demandId}`

查询家教需求详情，包含发布人、科目、年级、预算、状态和审核备注。

### `GET /api/catalog/tutors`

查询前台展示的已认证教员资料列表。

### `GET /api/catalog/tutors/{profileId}`

查询教员资料详情，包含教学经验、服务方式、文字简历和附件下载地址。

### `GET /api/catalog/tutors/{profileId}/resume-file`

下载教员已上传的简历附件，供教员本人和管理员审核时查看。

### `GET /api/catalog/tutor-users/{tutorUserId}/profile`

根据教员用户 ID 查询对应资料，供管理员从申请记录跳转查看教员资料。

## 5. 管理员接口

### `GET /api/admin/profiles`

查询教员认证资料列表。

### `POST /api/admin/{adminId}/profiles/{profileId}/audit`

审核教员认证资料。

### `GET /api/admin/demands`

查询家教需求列表。

### `POST /api/admin/{adminId}/demands/{demandId}/audit`

审核家教需求。

### `GET /api/admin/applications`

查询接单申请列表。

### `POST /api/admin/{adminId}/applications/{applicationId}/approve`

通过接单申请并生成订单。

### `POST /api/admin/{adminId}/applications/{applicationId}/review`

统一审核接单申请，`approved=true` 时通过申请并生成订单，`approved=false` 时驳回申请并记录驳回理由。

### `GET /api/admin/orders`

查询平台订单列表。

### `GET /api/admin/audits`

查询最近审核记录。

## 6. 统计接口

### `GET /api/dashboard`

返回平台首页统计数据，包括家长数、教员数、管理员数、开放需求数、待审核资料数、待处理申请数和服务中订单数。
