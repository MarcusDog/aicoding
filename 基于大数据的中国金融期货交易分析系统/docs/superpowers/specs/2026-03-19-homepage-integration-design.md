# 官网首页与系统展示页一体化设计

## 目标

将当前以答辩展示为主的首页升级为面向真实运营方的数据产品官网首页，并与系统展示页形成一体化产品体验。首页负责建立平台认知、展示核心能力、说明真实数据来源，并通过明确的页面动线将用户导入系统展示区。

## 已确认约束

- 产品形态：官网首页 + 系统展示页一体化完整产品
- 首页受众：真实运营方
- 风格方向：现代数据产品风格
- 首页主价值：整合真实公开数据，形成可视化分析平台
- 首屏主按钮：查看功能全景
- 结构方向：官网与系统双核，首屏即露出系统预览

## 页面结构

### 1. 官网首页

首页承担产品官网职能，结构如下：

1. 顶部导航
2. Hero 首屏
3. 平台能力模块
4. 功能全景模块
5. 系统展示预览模块
6. 真实数据来源与可信说明
7. 页脚

### 2. 系统展示页

系统展示页承接首页，不是独立断裂的另一套界面，而是作为产品内容的下半部分存在。展示模块包括：

- 市场总览
- 合约行情
- 趋势与波动
- 相关性与风险
- 数据来源说明

## 视觉设计

- 整体风格以现代数据产品为准
- 页面背景采用暖浅底与深色面板对比，形成“官网展示 + 分析系统”双层质感
- 首页强调大标题、强层级、较少解释性长段落
- 系统展示区保留数据看板属性，但视觉语言统一到产品首页中
- 页面避免“学校项目封面”气质，改为“分析平台官网”气质

## 内容映射

首页内容必须与开题报告功能对应：

- 真实数据接入：CFFEX、PBC、NBS
- 分析能力：趋势分析、波动率预测、相关性分析、VaR
- 展示能力：市场总览、合约行情、趋势与波动、风险分析、数据来源说明
- 可信能力：数据质量、健康状态、来源可追溯

## 交互要求

- 首页按钮支持跳转到“功能全景”和“系统展示区”
- 导航支持快速跳转不同产品区块
- Thesis 模式下仍保留截图能力
- 原有 API 与数据加载方式不变

## 实现策略

- 在现有 `app/streamlit_app.py` 上重构，而不是另起前端工程
- 保留现有真实数据读取与分析服务逻辑
- 将当前 `defense` 页面改造成正式首页 `home`
- 重新组织页面导航与内容函数
- 保留系统各分析页面，并统一文案和视觉层级

## 验收标准

- 首页视觉明确像正式数据产品官网
- 首页与系统展示区形成自然的一体化动线
- 开题报告核心功能均在系统展示区中可见
- 真实数据链路、接口、页面均可运行
- 截图脚本仍可输出有效页面截图
