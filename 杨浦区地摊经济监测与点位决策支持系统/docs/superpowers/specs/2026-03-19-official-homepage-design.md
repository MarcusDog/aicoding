# 正式产品首页设计

**目标：** 将现有论文/答辩原型收敛为“已进入系统后的正式首页”，服务真实运营方日常使用。

## 定位

- 首页类型：运营驾驶舱型
- 用户角色：运营人员、街区管理人员、业务分析人员
- 首屏目标：
  - 看当前风险态势
  - 看当前推荐点位
  - 快速进入具体模块继续处理

## 信息架构

- 顶部品牌区：平台名称、产品说明、数据状态、最近更新时间、导出入口
- 主导航：首页、推荐分析、投诉监测、人工标注、模型诊断、证据导出
- 首屏指标：投诉总量、定位覆盖率、候选点位数、高适宜点位数、高风险片区数、标注进度
- 主工作区：
  - 左栏：本周运营摘要、重点投诉类型、风险提示
  - 中栏：杨浦区热点与推荐点位地图
  - 右栏：Top3 推荐卡和快捷操作
- 第二层：模块入口卡
- 次级区：详细分析、原始数据预览、系统说明

## 视觉方向

- 风格：正式政务运营产品，而非论文演示稿
- 配色：深海军蓝、青绿色、橙色风险提示、浅色背景
- 表现：地图为中心，但保留运营卡片和导航层级

## 技术约束

- 保持 `app/main.py` 作为入口
- 不破坏现有数据处理、标注、导出、证据链功能
- 首页为主视图，其余模块在同页内切换显示
