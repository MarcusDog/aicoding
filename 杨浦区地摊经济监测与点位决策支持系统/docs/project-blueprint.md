# 项目蓝图

## 题目建议

基于多源时空数据的杨浦区地摊经济监测与点位推荐系统设计与实现

## 研究对象

杨浦区高校、商圈、地铁口与社区过渡地带的摊位活动监测与候选点位推荐。

## 研究问题

1. 如何把投诉数据、点位数据与时间特征统一到可计算的空间单元中。
2. 如何在真实数据有限的情况下构建可解释的点位适宜度排序。
3. 如何用一个轻量 Web 原型把推荐结果、风险提示和依据展示出来。

## 技术路线

1. 获取真实投诉数据和候选点位数据。
2. 做清洗、时间标准化、地理编码和空间网格化。
3. 构造投诉密度、位置邻近度、节假日、天气等特征。
4. 先使用规则评分，再使用逻辑回归等轻量模型做对照。
5. 在 Streamlit 中展示热点地图、Top3 推荐和解释图。
6. 用系统截图、测试记录、问卷或标注结果支撑论文。

## 需要尽快确认的事项

- 真实数据是否能拿到。
- 候选点位的空间范围是否固定。
- 是否保留“监测系统”题目，还是收缩为“决策支持原型”。
