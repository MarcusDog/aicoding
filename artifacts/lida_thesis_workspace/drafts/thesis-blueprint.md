# Thesis Blueprint

## Title

基于大数据的中国金融期货交易分析系统设计与实现

## Research Object

以中国金融期货交易所股指期货 `IF`、`IH`、`IC`、`IM` 为主要研究对象，结合中国人民银行和国家统计局公开发布的宏观金融数据，构建面向盘后分析的多源数据处理与可视化系统。

## Research Goal

围绕真实公开数据，完成一套可运行、可复核、可展示的中国金融期货交易分析系统，实现数据采集、入仓管理、趋势分析、波动率预测、相关性分析、风险价值计算和可视化展示，并在毕业论文中系统论证其设计思路、关键实现与测试结果。

## Technical Route

1. 从 CFFEX、PBC、NBS 获取真实公开数据并保留原始文件。
2. 对原始数据进行标准化、清洗和结构化存储，构建 DuckDB + Parquet 轻量数据仓。
3. 根据持仓量与成交量规则生成主力连续序列。
4. 基于收益率序列计算均线、回撤、滚动波动率、EWMA 波动率预测和历史模拟法 VaR。
5. 将期货收益与宏观指标按月对齐，构建相关性矩阵。
6. 通过 FastAPI 提供查询接口，通过 Streamlit 提供系统展示页面与论文截图资产。
7. 基于学校模板生成规范化论文文档，完成参考文献核验、目录更新和格式审计。

## Argument Line

论文论证主线不是简单的“做了一个系统”，而是证明如下观点：

- 真实公开数据能够支撑中国金融期货盘后分析系统的构建。
- 轻量数据仓架构可以满足毕业设计场景下的数据处理、风险分析与查询展示需求。
- 将主力连续、波动率预测、相关性分析与 VaR 集成到统一平台中，可以显著提升分析链路的一致性和结果可解释性。
- 通过模板化文档生成与图表导出，可实现系统研发成果向论文交付成果的可重复转化。

## Chapter Plan

1. 绪论
   - 说明研究背景、现实意义、国内外研究现状、研究内容与论文结构。
   - 重点论证金融期货盘后分析与大数据处理结合的必要性。
2. 相关技术与理论基础
   - 介绍股指期货、数据仓、DuckDB/Parquet、Spark local、FastAPI、Streamlit 等实现基础。
   - 给出收益率、年化波动率、EWMA、历史模拟法 VaR 与 Pearson 相关系数等公式。
3. 需求分析与数据资源设计
   - 从业务需求、功能需求、非功能需求三个层面界定系统目标。
   - 说明 CFFEX、PBC、NBS 三类数据源及数据表设计。
4. 系统总体设计
   - 说明系统总体架构、数据流、模块划分、接口设计和数据库设计。
   - 突出从采集层、存储层、分析层到展示层的链路闭环。
5. 系统详细实现与关键技术
   - 详细展开数据采集实现、主力连续构建、波动率预测、风险分析、接口封装和前端展示。
   - 插入代码、截图与系统页面说明。
6. 系统测试与结果分析
   - 给出测试环境、功能测试、数据质量测试和结果图分析。
   - 结合 2026-03-19 的最新分析快照解释 IF、IH、IC、IM 的风险与趋势状态。
7. 结论与展望
   - 概括研究结论、项目价值、局限性与后续扩展方向。
