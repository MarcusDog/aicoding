# 正式产品首页设计

## 目标

将当前偏功能演示的 [site.py](/C:/Users/lenovo/OneDrive/Desktop/毕业设计_4/基金融交易欺诈检测/app/site.py) 改造成正式产品首页，面向学校老师评审场景，兼顾产品感、技术可信度和系统体验入口。

## 用户定位

- 主要受众：学校老师 / 答辩老师
- 首页气质：正式、可信、现代、克制
- 页面定位：商业化产品展示首页，而不是后台监控页或论文封面页

## 设计方向

- 视觉方向：冷静科技型
- 主色：蓝白灰，少量青色点缀
- 页面节奏：先建立产品价值与可信度，再引导进入检测功能

## 页面结构

1. Hero 首屏
2. 能力概览区
3. 技术可信区
4. 风险识别流程区
5. 模型结果区
6. 体验入口区

## 功能要求

- 保留单笔检测、批量检测、运行概览三类交互入口
- 首页直接展示关键指标和实验图，而不是要求用户切换到答辩页
- 页面内容来源于真实训练产物与系统运行产物
- 对没有流式结果或模型文件的场景保持稳健降级

## 工程约束

- 基于现有 Streamlit 技术栈实现
- 不破坏现有答辩版 [demo.py](/C:/Users/lenovo/OneDrive/Desktop/毕业设计_4/基金融交易欺诈检测/app/demo.py)
- 把首页内容配置从页面渲染逻辑中拆分，降低后续继续调文案和布局的成本

## 验证要求

- 增加针对首页内容配置的自动化测试
- 保持现有 API / 评估测试通过
- 启动网站后能在浏览器正常访问并看到新的正式产品首页
