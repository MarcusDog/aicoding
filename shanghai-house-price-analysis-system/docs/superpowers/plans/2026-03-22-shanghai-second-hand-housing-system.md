# 上海市二手房价数据分析系统项目规划

> 基于开题报告《基于大数据的上海市二手房价数据分析系统设计与实现》整理，作为后续系统开发、论文撰写和答辩准备的统一执行版本。

1. `预期研究成果` 中出现了“网上电脑配件商城系统”，与本课题无关，必须删除并改为本项目表述。
2. `工作进度计划` 中出现了“协同过滤算法开发与调试”，与房价分析系统不符，必须替换为“数据分析模型/房价预测模型开发与调试”。
3. 开题报告未明确真实数据源、字段范围、预测目标、评价指标和系统部署方式，这些内容需在需求分析和论文正文中补齐。

## 2. 项目目标

完成一个可在 Windows 本机部署和演示的上海市二手房价数据分析系统，具备以下核心能力：

- 采集真实二手房相关数据
- 对数据进行清洗、存储和统计分析
- 可视化展示上海二手房市场特征
- 提供一个基础可解释的房价预测功能
- 支持答辩演示、截图、导出和论文材料整理

## 3. 项目范围

### 3.1 必做范围

- 上海市二手房挂牌或成交相关数据采集
- 数据清洗、字段标准化、去重、异常值处理
- MySQL 数据库存储
- 区域、面积、户型、价格、时间趋势等统计分析
- ECharts 可视化展示
- 基础预测模型
- 后台任务管理、日志查看、数据导出

### 3.2 不做范围

- Hadoop/Spark 真分布式集群部署
- 在线交易、支付、签约等业务系统
- 推荐系统、协同过滤、复杂深度学习方案
- 强依赖付费商业接口的实现路线

### 3.3 MVP 边界

MVP 只要求完成：

- 1 个主数据源
- 1 个补充数据源
- 采集到分析再到展示的完整闭环
- 1 个基础预测模型

## 4. 真实数据方案

### 4.1 推荐数据组合

主数据源：

- `上海网上房地产 / 上海市房地产交易中心`，用于官方口径的房产信息、公告、统计或二手房相关公开页面

补充数据源：

- `上海贝壳找房` 或 `上海链家`，用于挂牌房源属性样本
- `上海房天下`，用于补充挂牌页面样本
- `国家统计局 70 个大中城市住宅销售价格指数`，用于官方月度价格趋势

辅助数据源：

- `上海市公共数据开放平台`，用于行政区划、交通、公共服务等外部特征补充

### 4.2 数据源使用原则

- 官方数据用于论文论据、宏观趋势、政策影响分析
- 平台数据用于房源属性建模和微观样本分析
- 若某平台限制抓取，则保留为补充源，不作为唯一主源
- 严禁编造不存在的数据集或伪造成交数据

### 4.3 建议采集字段

- 房源标题
- 区县
- 板块
- 小区名称
- 总价
- 单价
- 面积
- 户型
- 楼层
- 朝向
- 装修
- 建成年份/楼龄
- 是否近地铁
- 挂牌时间
- 数据来源
- 抓取时间

### 4.4 风险说明

- 官方站点字段粒度可能不够细，不一定直接公开单套房源级明细
- 贝壳/链家/安居客/房天下存在反爬、登录、页面结构变动风险
- 平台“成交价”通常不完全开放，论文中需明确区分“挂牌价分析”和“官方指数分析”

## 5. 系统设计

### 5.1 核心模块

1. 数据采集模块
2. 数据清洗与预处理模块
3. 数据存储模块
4. 数据分析模块
5. 可视化展示模块
6. 房价预测模块
7. 后台管理与日志模块
8. 数据导出模块

### 5.2 推荐技术栈

- 后端：`Django 5 + Django REST Framework`
- 前端：`Vue 3 + Vite + Element Plus + ECharts`
- 数据处理：`Python 3.11 + pandas + numpy + scikit-learn + statsmodels`
- 抓取：`requests + BeautifulSoup`，必要时加 `Playwright`
- 数据库：`MySQL 8`
- 任务调度：`APScheduler`
- 环境：`Windows 10/11 + Node.js 20 + Python 3.11`

### 5.3 数据库草案

- `house_listing`
- `house_listing_snapshot`
- `community_info`
- `district_stat`
- `price_index`
- `crawl_task`
- `prediction_result`
- `export_log`
- `sys_user`、`sys_role`、`sys_log`（如需后台账号）
